# Security Policy

## Scope

This project provides read-only baseline checks and documentation for Raspberry Pi / PiCar-X style lab devices.

## Reporting Issues

Please do not publish exploit details in public issues. Open a private advisory where possible, or contact Musaab Hasan through:

- https://musaab.info/contact
- https://www.linkedin.com/in/engmusaabhasan/

## Design Boundaries

- The tool must not change system configuration.
- Checks should avoid collecting secrets, tokens, private keys or classroom learner data.
- Reports should focus on posture and configuration, not personal information.
- Findings should be framed as review items because lab environments may intentionally relax some controls for instruction.
